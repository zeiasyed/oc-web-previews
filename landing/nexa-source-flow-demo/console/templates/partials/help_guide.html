NexaFlow Help Digital guide

NexaFlow (NexaSource) moves structured eSource data from ClinSpark into your EDC without duplicate entry. Coordinators upload a subject roster, pick a visit and roster subset, and push mapped fields to Exxel EDC (demo) — with intelligent flagging when EDC already holds different values. This lab console demonstrates the full sync, review, and verification loop sponsors expect from modern eSource-to-EDC integration.

Four-step workflow

Upload → Select subjects → Write to EDC → Verify in Exxel EDC. Each step maps to a visible card in the console.

Flag-aware sync

When EDC already has data, NexaFlow surfaces side-by-side comparison so coordinators accept, edit, or skip field-by-field.

Fresh demo every visit

Opening this console resets mock EDC and clears prior sync state. Subjects are not pre-loaded — you start at step 1.

1 Quick start — 5-minute walkthrough

  1. Upload ClinSpark subjects — Click the teal Upload ClinSpark Subjects button. The activity log simulates an export pull; 45 synthetic subjects load into the roster grid.
  2. Choose study & visit — Confirm the study dropdown (NovaPharm Phase 1) and select a visit (e.g. Screening).
  3. Select subjects — Click subject chips to select individuals, use All/Clear, or click a saved Cohort pill (Cohort 1 / Cohort 2) for quick roster picks.
  4. Write to EDC — Click the green Write to EDC button. Progress bar and log show per-subject sync steps.
  5. Review flagged items — If EDC already had values, the Review flagged items modal opens automatically. Accept or edit flagged rows, then click Write to EDC in the modal to commit resolutions.
  6. Verify — Click Open EDC to verify → to inspect synced CRFs in Exxel EDC.
Presenter tip: Deliberately mention that upload is manual step 1 — it mirrors real ClinSpark API/export integration and shows coordinators control who syncs before data moves.

2 Upload ClinSpark subjects

Before any sync, you must pull the subject roster from ClinSpark. Until upload completes:

  • The subjects card shows a locked message — no chips visible
  • Write to EDC stays disabled
  • The workflow step indicator highlights step 1 · Upload

After upload, the button hides and a green badge shows the count (e.g. “✓ 45 subjects loaded”). The activity log lists each subject ID as it arrives — expand the log for full detail.

In production, NexaFlow connects to ClinSpark via sponsor-configured API or scheduled export. This demo simulates latency and log output for realism.

4 Write to EDC (sync engine)

Write to EDC pushes ClinSpark-mapped values for the selected subjects and visit into Exxel EDC. During sync:

  • Header status shows Writing to EDC…
  • Progress bar advances per subject and form step
  • Activity log streams detailed steps (subject ID, form names, field counts)

When sync completes without flagged items, a success card summarizes entered vs. skipped fields. The main Write button disables after a successful full write until you change visit/subjects or click Reset.

Changing visit or subject selection after a write re-enables the button so you can sync a different roster slice.

5 Review flagged items

When mock EDC already contains values that differ from ClinSpark source, NexaFlow opens the Review flagged items modal automatically. Each row shows:

  • Subject, form, and field name
  • ClinSpark (source) value vs. current EDC value
  • Status indicator — focus on rows marked ✗

For each flagged item you can:

  • Accept — Use the ClinSpark value (overwrites EDC on commit)
  • Edit — Enter a corrected value before accepting

Click Write to EDC in the modal footer to commit all accepted/edited resolutions. After a full write, the button hides and messaging confirms everything is synced.

Minimize (−) keeps the review panel accessible while you check Exxel EDC side-by-side. Print exports the review table for audit discussions.

Partial saves keep the Write button visible — only a complete resolution of all open flagged items counts as “all written.”

6 Mock EDC verification

Open EDC to verify → opens Exxel EDC (demo) with session token, first synced subject, and selected visit pre-loaded.

  • Browse forms in the EDC form navigation tree
  • Confirm field values match accepted ClinSpark data
  • Cross-check flagged fields you resolved in the review modal

Empty forms mean no mapped data was written for that subject/visit/form — not a display bug if sync skipped those fields.

9 Frequently asked questions

Why aren’t subjects loaded when I first log in?
By design — upload is step 1 of the workflow demo. Each page load resets to a clean state so every presentation starts the same way.
What’s the difference between Save and Write to EDC in the review modal?
Accept/Edit stages your decisions locally. Write to EDC in the modal commits resolved values to Exxel EDC — matching production audit separation.
Why is Write to EDC disabled?
Upload subjects first, then select at least one subject chip. It also disables after a successful write until visit/subjects change or you Reset.
Why does mock EDC show empty fields I thought were synced?
Check visit/subject match, flagged-item review completion, and form mapping. Some demo flagged items are on specific forms (e.g. PK collection vs. labs).
Does NexaFlow replace ClinSpark?
No — it complements eSource. ClinSpark remains the capture UI; NexaFlow eliminates re-keying into EDC.
Which EDCs are supported?
Major sponsor EDCs (Veeva, Oracle, Castor, and others) via Nexa Bus — same automation layer as NexaDirect.
