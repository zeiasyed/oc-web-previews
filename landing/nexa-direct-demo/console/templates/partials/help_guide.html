NexaDirect Help Digital guide

NexaDirect automates the path from scanned paper source to your electronic data capture (EDC) system. This lab console watches a scanner inbox folder, extracts CDASH-mapped fields from PDFs, auto-writes high-confidence values, and routes uncertain extractions to a structured review queue — so coordinators spend less time on duplicate entry and more time with patients.

What you’re seeing

A fully interactive demo with synthetic study data, Exxel EDC demo, and a simulated scanner inbox of 24 PDFs.

What NexaDirect does in production

OCR + field mapping, confidence scoring, audit trail, and direct writes to sponsor EDCs via Nexa Bus.

Fresh start every visit

Each time you open this console, demo data resets automatically so you always begin with a clean inbox and empty EDC.

1 Quick start — 5-minute walkthrough

  1. Confirm study — The study dropdown at the top selects the active protocol (demo: NovaPharm Phase 1).
  2. Preview source PDFs — Scroll to Scanner Inbox and click any filename (e.g. 0102_AE.pdf) to open an in-app PDF preview. Or use Scanned Inbox Folder to reveal the folder in Windows Explorer.
  3. Run extraction — Click Write to EDC in the workflow card. NexaDirect processes all inbox PDFs sequentially. Watch the progress bar and activity log on the right.
  4. Review flagged fields — Fields below the confidence threshold appear in Review flagged items. Click View scan to compare against the PDF, edit values if needed, then Approve each row.
  5. Commit staged approvals — Approved values are staged only until you click Write to EDC in the review section (yellow banner explains this). That second write pushes approved values to the mock EDC.
  6. Verify — Click Open EDC to verify → at the bottom of the review section (or the header button) to open Exxel EDC with the synced subject and visit pre-selected.
Presenter tip: Start by clicking one inbox PDF so the audience sees the scanned form, then run Write to EDC to show automation vs. manual review in one pass.

3 Write to EDC (main process)

The primary Write to EDC button launches batch extraction across all inbox PDFs for the selected study. For each file, NexaDirect:

  • Parses the PDF and maps extracted text to CDASH field definitions configured for that form
  • Scores each field for extraction confidence (0–1)
  • Auto-writes fields at or above the confidence threshold (default 0.85 in Study Setup)
  • Flags lower-confidence or ambiguous fields for human review

The status pill in the header shows Processing… while running and Complete when finished. The activity log summarizes totals: files processed, auto-written field count, and flagged count.

Two “Write to EDC” buttons: The top workflow button runs extraction. The review-section button commits approved flagged values after you’ve staged them — they are different steps in the pipeline.

4 Review flagged items

When extraction confidence is below threshold, fields appear here with the issue type (e.g. low_confidence), extracted value, and confidence score.

  • View scan — Opens the source PDF side-by-side context so reviewers can validate against the paper form
  • Approve — Accepts the extracted value (or your edited value) and stages it for EDC write
  • Edit — Click the value cell to correct before approving
  • Undo approve — Available on staged rows if you need to revert before committing

Form names display in full in the review table (e.g. “Adverse Events” rather than “AE”) so clinical reviewers see familiar CRF language.

After all flagged rows are approved, click Write to EDC in the review banner to commit staged values. When everything is written, the review write button hides and Open EDC to verify → appears at full width.

Rows turn green when staged. The yellow banner reminds you that staging ≠ EDC commit until you explicitly write.

5 Mock EDC verification

Open EDC to verify launches Exxel EDC in a new tab, pre-authenticated with a session token. You land on the Case Report Form for the first synced subject and default visit.

  • Navigate subjects and visits using the Exxel EDC shell
  • Confirm auto-written and approved fields appear in the correct forms
  • Empty forms indicate no data was synced for that subject/visit/form combination

In production, NexaDirect writes through validated API connectors with full audit trail — this demo uses a local SQLite-backed mock for speed and safety.

6 Study setup (advanced)

Expand Study Setup to configure site-level and form-level settings:

Site & inbox

Study name, default visit, site ID, scanner inbox path, and auto-write confidence threshold.

Forms & fields

Enable/disable CRFs, edit display titles, map PDF file codes, import CDASH Excel definitions.

Changes persist for the demo session. Use Reset to return EDC, inbox statuses, and review queue to a clean state without reloading the page.

9 Frequently asked questions

Why does the review banner say values aren’t written yet?
Approving a flagged field only stages it. A separate Write to EDC in the review section commits staged values — mirroring production workflows where review and commit are distinct audit events.
Why is a form empty in mock EDC after sync?
No fields mapped to that form for that subject/visit, or all fields were flagged and not yet approved/committed. Check the activity log and review queue.
Can I re-run Write to EDC?
After a successful run, use Reset for a clean slate, or process again — already-done files retain status until reset.
Does this connect to a real EDC?
This lab uses Exxel EDC. Production NexaDirect connects via Nexa Bus to sponsor-configured EDC endpoints.
What EDCs are supported?
Major sponsor EDCs (Veeva, Oracle, Castor, and custom APIs) — scoped per deployment.
